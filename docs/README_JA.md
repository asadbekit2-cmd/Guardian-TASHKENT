## 1. 概要

**Guardian – エコ・ディフェンダー** は、環境保護をテーマにした、モバイルファーストのブラウザベースのマッチ3パズルゲームです。  
プレイヤーはAR（拡張現実）風スキャナーで周囲を探索し、エコモンスター（汚染の化身）をロックオンして、8×8のマッチ3グリッドで戦います。

### 主な機能

| 機能 | 説明 |
|---|---|
| 📱 モバイルシミュレーター | デスクトップブラウザ上でスマートフォンのフレームUIとして動作 |
| 🔊 シンセサウンド | Web Audio APIによる音声効果（音声ファイル不要） |
| 🕹️ マッチ3エンジン | スワップ・マッチ・カスケードを備えた8×8グリッド |
| 🏆 リーダーボード | **E → D → C → B → A** のランク昇格システム |
| ⚙️ 設定パネル | 音楽・音量トグルとドラッグスライダー |
| 🌊 ARスキャナーフロー | アニメーション付きビューファインダー → ターゲットロック → バトルシーケンス |
| ✨ パーティクルFX | ボタンクリック時のリップル効果と粒子爆発エフェクト |

## 3. ゲーム画面とフロー

このゲームはシングルページアプリケーション（SPA）です。各画面の表示は `.active` CSSクラスの付け外しで制御されます。

```
ホーム画面
    │
    ├──[設定ボタン]─────► 設定モーダル（オーバーレイ）
    ├──[バッジボタン]───► リーダーボードモーダル（オーバーレイ）
    └──[スタートボタン]─► ARスキャナービュー
                              │
                         [スキャンボタン]
                              │
                         デストロイビュー（ターゲットロック）
                              │
                         [デストロイボタン]
                              │
                         マッチ3バトル画面
                              │
                    ┌─────────┴─────────┐
               [勝利: HP=0]      [敗北: タイマー=0]
                    │                   │
             ビクトリーモーダル    デフィートモーダル
                    │                   │
              [続けるボタン]     [もう一度ボタン]
                    └─────────┬─────────┘

```

## 4. コアモジュール（JavaScript）

すべてのモジュールは `game.js` 内の `DOMContentLoaded` イベントで初期化されます。

### `initClock()`
`setInterval` を使用して毎秒ステータスバーの時計を更新します。  
**対象要素:** `#clock`

---

### `initAudioSynth()`
グローバル関数 `window.playSynthSound(type)` を登録します。  
**Web Audio API** を使用 — 外部音声ファイルは不要です。

| サウンドタイプ | トリガー | 説明 |
|---|---|---|
| `'start'` | スタートボタン | 上昇する三角波のエネルギッシュなスイープ |
| `'click'` | 任意のボタンタップ | ソフトなサイン波のタップ音 |
| `'hover'` | ボタンへのマウスオーバー | 900Hzの軽やかなチャイム音 |

> **注意:** AudioContextはブラウザの自動再生ポリシーに準拠するため、最初のユーザー操作時に遅延生成されます。

---

### `initClickEffects()`
全ボタン押下時にクリックリップルとパーティクルバースト効果を追加します。

- **リップル:** 白い円がクリック地点から広がりフェードアウトします。
- **パーティクル:** 6つの色付き点が外側に散らばります（グリーン `#38ef7d`、シアン `#11998e`、ティール `#78ffd6`、ゴールド `#f6d365`）。

---

### `initHomeControls()`
ホーム画面の3つのボタンを設定します。

| ボタン | アクション |
|---|---|
| `#start-game-btn` | `'start'`音を再生 → `#scanner-view` へ遷移 |
| `#settings-btn` | `'click'`音を再生 → `#settings-modal` を開く |
| `#badges-btn` | `'click'`音を再生 → `#leaderboard-modal` を開く |

---

### `initSettingsModal()`
**設定**オーバーレイを管理します。
- **音楽トグル** (`#modal-music-btn`): バックグラウンドミュージックのミュート/ミュート解除。
- **音量トグル** (`#modal-volume-btn`): SEFXのミュート/ミュート解除。
- **音量スライダー**: ドラッグ＆ドロップ + タッチ操作対応。`currentVolumeScale`（0〜1）を更新し、全ゲインノードをスケール調整。

---

### `initScannerView()`
**ARスキャナー**画面を制御します。
1. 閉じるボタンでホームに戻る。
2. スキャンボタン（`#scan-trigger-btn`）がノコギリ波のオーディオスイープを起動し、ビューファインダーのステータステキストを更新し、**1,500ms後**にデストロイ画面へ遷移。

---

### `initDestroyView()`
**ターゲットロック**画面を制御します。
1. 閉じるボタンでホームに戻る。
2. デストロイボタン（`#destroy-trigger-btn`）が二重オシレーターの爆発音を鳴らし、HUD上に20個のパーティクルを生成し、**750ms後**にマッチ3へ遷移して `startMatch3Game()` を呼び出す。

## 6. UIコンポーネントとCSS

### レイアウトシステム
ゲームは**デバイスラッパー**アプローチを使用し、デスクトップ上でスマートフォンをシミュレートします。

```
.device-wrapper       → デスクトップ上でスマートフォンを中央に配置
  .device-screen      → 表示されるスマートフォン画面（固定アスペクト比）
    .status-bar       → 信号/時計/バッテリーアイコン
    .game-screen      → 各画面コンテナ（デフォルト非表示）
    .modal-overlay    → 設定とリーダーボードのオーバーレイ
```

### 主なCSSアニメーション

| アニメーション名 | 使用箇所 | 効果 |
|---|---|---|
| `scanLine` | `.scanner-laser-line` | 緑色のスキャンレーザーが上から下へスイープ |
| `sonarSpin` | `.hud-sonar-line` | ターゲットHUD内のソナースイープ回転 |
| `ringPulse` | `.rank-pulse-ring` | アクティブなランクバッジに輝くリングが脈打つ |
| `starPop` | `.vstar` | 勝利画面で星が順番にポップイン |
| `victoryGlow` | `.victory-screen` | 勝利時の画面全体へのゴールデングロー |

## 7. オーディオシステム

オーディオエンジンは**Web Audio API**のみを使用します。全サウンドは実行時に合成され、外部ファイルは不要です。

### サウンドアーキテクチャ

```
オシレーター（タイプ: sine/triangle/sawtooth）
    │
    ▼
ゲインノード（AudioParamランプによる音量エンベロープ）
    │
    ▼
AudioContext.destination（スピーカー出力）
```

### 音量コントロール
`currentVolumeScale`（グローバル変数、0〜1）がすべてのゲイン値に乗算されます。  
UIスライダーの0〜100%の位置がこのスケールに直接マッピングされます。

## 8. マッチ3バトルエンジン

コアゲームループは以下の関数で管理されます。

### `startMatch3Game()`
- HPを `100`、タイマーを `45秒` にリセット。
- `monstersList` からランダムにモンスターを選択。
- `generateStartBoard()` → `renderBoardHTML()` → `startMatch3TimerLoop()` を呼び出す。

### ボードの状態変数

| 変数 | 説明 |
|---|---|
| `match3Board` | アセットインデックス（0〜4）の8×8二次元配列 |
| `selectedCell` | 現在選択中のタイル `{row, col}` または `null` |
| `isBoardLocked` | アニメーション中の操作をブロック |
| `match3TimeLeft` | 45から0へのカウントダウン |
| `monsterHP` | 100から始まりマッチしたタイルで減少 |

### ゲームループ

```
プレイヤーがタイルAを選択
    │
プレイヤーがタイルB（隣接）を選択
    │
AとBをスワップ
    │
マッチをチェック（行または列に3つ以上）
    ├── マッチなし → スワップを元に戻す
    └── マッチあり
            │
        マッチしたタイルを削除
            │
        モンスターHPにダメージ
            │
        タイルを落下させ上から補充
            │
        新しいマッチを確認（カスケード）
            │
        勝利条件をチェック（HP≤0） ──► 勝利
```

### ダメージの計算式
マッチした各タイルはモンスターに **5 HP** のダメージを与えます。  
マッチ3 = **15 HP**、マッチ4 = **20 HP**、マッチ5 = **25 HP**

### タイマー
`setInterval`（1,000msティック）で45秒のカウントダウンが動作します。
- `0`になると → `triggerDefeat()`（敗北）
- HP ≤ 0になると → `triggerVictory()`（勝利）

## 10. 起動方法

このプロジェクトはビルドステップ不要の**純粋なHTML/CSS/JS**プロジェクトです。

### オプション A — 直接開く
```bash
# プロジェクトフォルダから:
open index.html
# またはファイルマネージャーで index.html をダブルクリック
```

### オプション B — ローカルHTTPサーバー（推奨・CORSエラーを回避）
```bash
# Python 3
python3 -m http.server 8080

# ブラウザで以下を開く:
http://localhost:8080
```
